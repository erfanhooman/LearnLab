Asymptotic Notations

The Big-O Notation

Big O notation is a mathematical notation used to describe the upper bound on the time complexity of an algorithm or the worst-case time it takes for an algorithm to run

Proving the Big-O is like to find existence of c and n0: O(g(n)) = f(n) : f(n) =< c.g(n) for all n > n0

The Big-Ω Notation

Big O notation is a mathematical notation used to describe the lower bound on the time complexity of an algorithm or the best-case time it takes for an algorithm to run

Proving the Big-Ω is like to find existence of c and n0: Ω(g(n)) = f(n) : f(n) => c.g(n) for all n > n0

The Θ Notation

Θ notation is a mathematical notation used to describe the tight bound on the time complexity of an algorithm or the best-case time it takes for an algorithm to run

Proving the Θ is like to find existence of c1, c2 and n0: Θ(g(n)) = f(n) : c1.g(n) <= f(n) <= c2.g(n) for all n > n0

The Little-o Notation

little O Notation is same as the Big O with different that unlike the big-O, little-O is used to denote a upper bound that is not asymptotically tight

Proving the Little-O is like to find existence of n0 for any c: O(g(n)) = f(n) : f(n) < c.g(n) for all n > n0

The Little-Ω Notation

little Ω Notation is same as the Big Ω with different that unlike the big-Ω, little-Ω is used to denote a lower bound that is not asymptotically tight

Proving the Little-Ω is like to find existence of n0 for any c: O(g(n)) = f(n) : f(n) > c.g(n) for all n > n0
